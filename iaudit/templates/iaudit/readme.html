Osquery Rollout FAQs:-

    Q.1 What is osquery?
    Osquery is cross platform operating system instrumentation framework. For all practice purposes is a small agent that sits on a machines and exposes key matrices related to system performance, security and compliance. You can read more about it here. With help of osquary we will be able to get a better visibility of our End User Computing security and performance analytics.

    Q.2 Why now?
    Currently we don’t have any capability to run any sort of org wide audits on our End User Machines. This puts us at a risk of not being able to take actions proactively. This has specially become even harder ever since we started working remotely.

    Q.3 What data is collected?
    You can view all of the supported tables and schema information: running processes, user logins, loaded kernel modules, open network connections, browser plugins, hardware events, file hashes, sockets, mounts, ports, storage volumes, packages etc. You can refer to the latest documentation to see all the tables that it supports

    Q.4 How is this not spying?
    There are no explicit privilege escalation methods built into osquery. OSQuery does not log any of the information anywhere. All it does is query the data that already exists in deep corners of your system’s config files, on the fly. It also makes a clear distinction between what data can be queried and makes sure it never invades user privacy. OSQuery is a reputed open source project that’s part of Linux Foundation now.

    In their own words: ”The osquery community respects developer and user privacy! We include a "non-goal" of exposing sensitive information like browsing history within tables. The osquery tools include 0 callback requests and 0 auto-updating, auto-diagnostic capabilities.” In their own words

    Q.5 Where is the osquery road map?
    We would initially be using OSQuery of general End User Inventory logging, running proactive IT audits and End Point Security Incident management. We later plan to use it for DLP but that’s a long way ahead.
